# Legal & Compliance Disasters Checklist

## Privacy violations
- [ ] Cookie consent banner where required
- [ ] Consent-based tracking (no silent tracking)
- [ ] No sharing user data without consent / lawful basis
- [ ] Data Processing Agreements (DPAs) with subprocessors
- [ ] Honor opt-out / “Do Not Sell/Share” where required
- [ ] Account deletion exists (self-serve) + verified
- [ ] User data export exists (self-serve) + verified
- [ ] Retention policy exists + automated deletion runs
- [ ] Data minimization
- [ ] Anonymization/pseudonymization for analytics
- [ ] No PII in logs, URLs, analytics payloads
- [ ] Third-party event payloads scrubbed/redacted
- [ ] Breach notification plan + timeline procedure
- [ ] DPO/Privacy contact process (when required)
- [ ] COPPA/age gate if minors possible
- [ ] CCPA/CPRA rights flows (access/delete/opt-out)
- [ ] ROPA maintained

## Terms of Service issues
- [ ] ToS exists (written for your product)
- [ ] Acceptance mechanism (checkbox + timestamp + version)
- [ ] ToS updates notify users + re-accept when material
- [ ] Liability limits and disclaimers fit your risk
- [ ] Dispute resolution + jurisdiction correct
- [ ] IP terms clear (your IP vs user content)
- [ ] Prohibited use + Acceptable Use Policy exists
- [ ] Termination and suspension terms exist

## Compliance violations (only if in scope or promised)
- [ ] PCI posture correct (no card storage; processor only)
- [ ] HIPAA controls + BAA (if PHI)
- [ ] SOC 2: don’t claim it without evidence
- [ ] Audit logs for required controls
- [ ] SLAs match reality
- [ ] Data residency commitments are real
- [ ] Vendor management program exists
- [ ] Accessibility (WCAG/ADA/508) addressed

## Licensing / IP disasters
- [ ] Dependency licenses tracked (SBOM)
- [ ] OSS notices included where required
- [ ] No GPL/AGPL surprises in proprietary repos
- [ ] Fonts licensed
- [ ] Stock photos/media licensed + receipts stored
- [ ] Third-party code attribution correct
