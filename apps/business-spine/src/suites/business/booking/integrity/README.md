BOOKING INTEGRITY ADD-ON
========================

This pack closes the "booking app edge-case" gaps that cause hiccups:
- policy engine (cancel/no-show/late)
- dispute workflow + evidence
- conflict/race-condition hardening
- provider + client reliability scoring (internal)
- immutable booking confirmations + versioning
- timezone/DST sanity layer
- refund leakage detection
- capacity/burnout protection rules
- booking simulation mode (admin)

Snap-in targets:
- booking spine
- payments/payouts modules
- admin/ops spine
- notifications layer
