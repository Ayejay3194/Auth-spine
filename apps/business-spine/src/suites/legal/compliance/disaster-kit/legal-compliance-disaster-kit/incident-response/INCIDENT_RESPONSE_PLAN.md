# Incident Response Plan (SaaS)

## Severity levels
- SEV0: confirmed data breach / widespread outage
- SEV1: major outage or suspected breach
- SEV2: partial outage / contained issue
- SEV3: minor issue

## Roles
- Incident Commander (IC)
- Security Lead
- Ops Lead
- Comms Lead
- Legal/Privacy Lead

## Workflow
1. Declare incident + severity
2. Contain (disable keys, block IPs, rotate secrets)
3. Preserve evidence (logs, snapshots)
4. Eradicate root cause
5. Recover (restore service, validate)
6. Postmortem + action items
